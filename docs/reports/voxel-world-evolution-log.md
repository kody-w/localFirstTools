Continuous 8-strategy improvement cycles. Each cycle: 8 strategy analyzers → consensus top 3 → implement → verify → deploy to GitHub Pages. Started 2026-06-12, running ~24h or until stopped.

## Cycle 0 — Baseline (2026-06-11)

Field notes from the launch-day twin fleet (already fixed — do not re-propose): PeerJS default broker fallback (&b= in link), per-tab rappid, action-JSON string coercion, spawn-inside-stone, void rescue, safe tp snap, unloaded-chunk pathing fence, phantom headless players suppressed.

Standing wishes from twin pilots (fair game for future cycles): block-break progress stages; richer observations for LLM brains (cheaper, more spatial); bot memory across saves; waypoints/pings; dig is async with no completion signal in the command form; map command shows terrain only at surface level (caves invisible).

## Cycle 1 — 2026-06-12

Implemented (file now ~5,440 lines; both inline scripts pass `node --check`; no duplicate top-level declarations; no missing element-id references):

1. **Hold-to-mine with crack stages + dig events.** Local-player breaking is now damage-accumulation (`breakState`, hardness table: Leaves 0.18s / Sand 0.30 / Grass+Dirt 0.35 / Wood 0.6 / Stone 0.9) with a 4-stage crack overlay (atlas tiles 9-12, stroked on transparent; one persistent 1.004-scaled box mesh, UVs touched only on stage change, skipped under headless). Release/look-away decays at 3×; completion runs the byte-identical old path (`setBlock` → sparse-diff save → neighborhood `edit` op). Touch: stationary hold past ~280ms mines continuously; quick tap applies `min(1, 0.3/hardness)` so leaves/dirt still one-tap. Bots (`bot.dig`) and `vox dig` keep instant single-call semantics; the ring buffer now gets `dig_progress` (quarter-stage transitions) and `dig_complete` (player, bot, and vox-command paths) — closes two standing wishes.
2. **Adaptive resolution governor + F3 instrumentation.** `renderScale` 0.6–1.0 in 0.1 steps applied as `basePR (= min(DPR,2)) * renderScale`; rides the existing 1s fps tick — drop after 2s under 45fps, raise after 4s over 55, max one step/second, quiet for 3s post-boot / while loading / while panels are open; never touches renderDist or NearestFilter. 'Auto quality' checkbox in settings (default ON for touch, OFF desktop) persisted in the settings blob (`autoQuality`); unchecking snaps back to 1.0. F3 HUD gains frame ms (avg), pixel ratio (eff), mesh ms (last, timed around `buildChunkMesh`); `vox status` now reports `renderScale`/`autoQuality`. Render call additionally guarded by `!document.hidden`.
3. **First-90-seconds onboarding coach.** `voxel-world-onboard-v1` key gates a 5-step bottom-center hint card (LOOK → MOVE → BREAK → PLACE → TOWER, IS_TOUCH-branched copy, only Skip takes pointer events) that ALSO runs for scan-to-join QR arrivals — the previously zero-guidance path. Counters live at the real action sites (break completion, `doPlace` success) and count from session start so out-of-order actions retroactively clear steps; hidden (never advanced) while panels/chat/terminal are open; suppressed under `?headless=1`/`?console=1`; private-mode degrades to once-per-session; coach replaces the auto help modal on first run and sets HELP_KEY on completion/skip. Joiners also get a "You're in <host>'s world" toast (host name from the snapshot's first roster entry).

Cycle 1 fixer pass (post-verification, same day): two real regressions fixed.
(a) Touch quick-tap math was broken: 0.3/0.35 = 0.857 meant grass/dirt never one-tapped, and 3/s decay between taps made stone un-tappable. Fix: tap constant bumped to 0.36 (grass/dirt/sand/leaves = one tap, wood 2, stone 3) plus a 400ms post-tap decay grace window (`lastTapT` guard in `decayMineDamage`) so consecutive taps actually accumulate. Hold-to-mine, bot dig, and vox dig untouched.
(b) The `!document.hidden` render guard was dead code for the hidden-tab promise — rAF stops entirely in hidden tabs, freezing twins/bots/broadcasts. Fix: a `hiddenTickTimer` setInterval (250ms, mirroring the `voxSetHeadlessTimer` pattern) drives `gameTick` while the tab is hidden and non-headless, cleared on return to visible. Note: browsers throttle background timers to >=1s, so hidden ticking is coarse (dt clamp bounds each step) — a Blob-worker clock would beat throttling if a future cycle needs full-rate hidden ticking.
(c) Interaction bug between (b) and the governor: hidden ticks render zero frames but still feed the fps accumulator (dt clamped to 0.1 → fpsVal reads as a constant 10), and `qualityGovernorTick` had no hidden guard, so with autoQuality on a hidden tab ratcheted renderScale to the 0.6 floor and the user came back to ~16-20s of blur. Fix: `|| document.hidden` added to the governor's pause condition (resets both streaks), and the visibilitychange return-to-visible branch now flushes `fpsFrames/fpsAcc/fpsLowStreak/fpsHighStreak` so the first visible 1s window (F3 HUD, `vox status`) measures only real rendered frames.

Field notes for future cycles:
- Crack overlay is local-only cosmetic v1 — remote players do not see partial mining progress. A `dig_progress` op on the wire (throttled like `pos`) would fix it if anyone cares.
- Desktop quick-click no longer breaks anything instantly (even leaves need 0.18s of hold) — intentional per spec; if playtests hate it, give desktop click the same `min(1, 0.3/hardness)` tap chunk as touch.
- Touch hold-to-mine starts at 280ms; a tap landing in the 280–350ms window gets both the held damage and the tap chunk (tiny double-count, accepted so quick taps can never fail to break leaves).
- The governor reads `fpsVal` (already a 1s average) with streak counters rather than a separate rolling buffer — same hysteresis, less state. Headless is a hard no-op (the 20Hz headless tick would otherwise read as "low fps" and pointlessly downscale).
- Existing players (HELP_KEY set, onboard key unset) will see the coach once after this deploy — accepted per spec; it self-dismisses in ~90s of normal play and Skip is permanent.
- Standing wishes still open: richer/cheaper LLM observations; bot memory across saves; waypoints/pings; map shows surface only (caves invisible).

### PRIORITY-SECURITY for cycle 2 (consensus chair: this MUST take one of the three slots)
Verified by cycle-1 devils-advocate: imported/persisted tether+cloud twins auto-start their network loop — world import (~line 1852) and boot re-hatch (~line 4036) call vfStartTwinLoop, which POSTs full world observations to the twin's arbitrary `url` (~line 3752) with no validation or consent. A malicious world-export JSON can therefore exfiltrate gameplay data to an attacker URL the moment it is imported. Fix: imported/re-hatched network twins land PAUSED with a visible badge; resuming requires an explicit user click backed by a per-URL allowlist persisted in localStorage (localhost defaults allowed); never auto-start a network twin whose URL has not been approved on this device.

## Cycle 2 — 2026-06-12
(The orchestration script labeled this run "cycle 1" of a fresh loop; in this log's numbering it is cycle 2 — it implements exactly the cycle-2 slate mandated above. File now ~5,940 lines; both inline scripts pass `node --check`; no duplicate top-level declarations; no missing element-id references; 48/48 functional checks pass in a Node harness run against the shipped code extracted verbatim.)

1. **Consent gate + import validation for network twins (the PRIORITY-SECURITY mandate).** `isAllowedTwinOrigin(url)` (localhost/127.0.0.1 any port always allowed; otherwise a persisted origin allowlist at `voxel-world-twin-url-allow-v1`, try/catch everywhere, storage failure degrades to localhost-only) now gates `vfHatchTwin`: approved origin → live; `fromImport` → paused with an amber badge, zero prompts, zero network; interactive hatch → one `window.confirm` showing the exact origin, accept persists. All three import funnels (boot re-hatch, world import, vox import) route through `vfImportRoster`, which also hard-drops malicious roster records via `validateTwinRecord` (no usable name / bad mode / non-http(s) or >200-char url — `javascript:` dies here) and coerces the rest; VF_MAX_BOTS enforced across existing+incoming; duplicate names skipped. Per-row Resume button (allow-or-confirm), `vox('resume <name>')` (never prompts; `origin not approved` otherwise), `vox('twinallow <origin>')` (operator full-trust). Roster JSON shape unchanged — 'paused' is derived at hatch, never persisted; `validateSave` additionally rejects a save whose `twins` is present but not an array.
2. **Sealed waypoint pings (op 'ping' on the rapp-twin-chat wire).** Middle-click (pointer-locked), a touch PING button, and `vox('ping [here|look|danger]')` ping the crosshair DDA target (fallback: 12 blocks along the view ray). Leaf→host sealed, host fan-out sealed, and — the trap the proposal warned about — `'ping'` added to the host hub-relay op allowlist so leaf→leaf propagates. Strict receive validation (finite+integer clamp, kind whitelist, 1s per-sender rate limit), one live marker per sender in `Neighborhood.pings` (replace on re-ping, 16 cap evicting oldest, 8s timestamp expiry per frame, full geometry/material/CanvasTexture disposal incl. `nbResetSession`). Marker = ring+name billboard sprite in the sender's color + a depth-test-free beam. Solo ping = personal local marker; headless skips all scene objects but still emits the `ping` vox event + sys chat line.
3. **Deterministic 3D cave carving + map y-slice.** `valueNoise3`/`fbm3` (ihash with a `Math.imul(y, 0x9E3779B1)` salt — pure in (seed,x,y,z)); `genChunk` carve pass after columns / before trees (edits overlay still last, so player edits win): STONE-only band `|fbm3(seed^0xCA7E, …)-0.5| < min(0.034, 0.018+depth*0.0008)`, oceans (h<SEA) skipped, y≤1 bedrock intact, WATER untouched; fbm3 sampled only on a WORLD-ALIGNED 4-block lattice (5×5×17/chunk) and trilinearly interpolated, so borders are seamless and chunk bytes are load-order independent (verified). `vox('map [r] [y]')` renders a horizontal slice at y (clamped 0..63) with '.' for air and entities marked only within ±2 of the slice; without y the output is byte-identical to before (diffed against git HEAD's implementation in the harness — text and return JSON byte-equal). Return adds `slice` key only in slice mode.

**Spec deviations (deliberate, measured):**
- The proposal's `t ~0.06` band measured 25-37% of underground stone carved (a value-noise fbm clusters near 0.5, so the band catches far more volume than intended). Shipped `0.018+depth*0.0008 (cap 0.034)` → 7.5-18% across 6 seeds, Minecraft-ish density.
- "Cave mouths appear where caves intersect exposed hillside/cliff stone" is unsatisfiable here: this terrain's adjacent-column height delta NEVER exceeds 1 (measured), and every column wears a 4-block non-stone cap, so a stone-only band can never daylight. Shipped a separate sparse deterministic mouth pass (`caveMouthAt`, p=0.0045/column): hillside grass columns only (never beaches/flat tops/under trees), and only where `caveFieldAt` — the same lattice-interpolated field the carve pass uses — CONFIRMS a cave 4-7 blocks below (no blind pits). Mouth shafts are margin-scanned like trees so they cross chunk borders seamlessly. Measured: 2-9 mouths within ~96 blocks of spawn (seed 1337 nearest ≈101), zero flat-top holes on every seed tried.
- Carve cost measured ~0.06ms/chunk in Node (~0.24ms at 4x throttle) — well inside the 8ms streaming budget.
